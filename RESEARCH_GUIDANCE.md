# 行研建议（Research Guidance）模块说明

## 模块定位声明
- 行研建议 ≠ 投资结论。
- 行研建议 ≠ 风险评级。
- 本模块的唯一作用：提示哪些财务变化“值得被写进研究报告”，帮助行研人员确定研究报告的切入方向，而非给出结论或投资决策。

## 数据依赖契约
- 行研建议只能基于 Logic 层已输出的数据结构进行解读与引用。
- 不允许直接访问 Parser 的原始数据。
- 不允许在建议层补算或推导任何新指标，仅可复用 Logic 层提供的现有指标与解释结果。

## 建议的结构化分类
- 盈利质量相关建议
  - 关注指标类型：利润构成、现金流匹配度、毛利率/净利率趋势、非经常性损益占比等。
  - 研究意义：帮助识别盈利的可持续性、利润质量及其波动来源，为报告中的“盈利质量分析”提供切入点。
  - 说明：此类建议仅指示需要关注的现象与指标，不提供正/负面结论。
- 财务结构 / 负债相关建议
  - 关注指标类型：资产负债率、短期/长期债务结构、利息保障倍数、资本开支与融资匹配度等。
  - 研究意义：帮助发现偿债能力与资本结构变化对企业风险与资金成本的影响，为“财务结构分析”提供切入点。
  - 说明：此类建议仅提示值得调查的变化，不给出风险评级或投资观点。
- 经营稳定性相关建议
  - 关注指标类型：收入分布、客户/供应商集中度、存货与应收账款周转、经营现金流波动等。
  - 研究意义：帮助识别经营韧性与业务持续性的潜在影响因素，为“经营稳定性讨论”提供切入点。
  - 说明：此类建议仅为研究起点，不评估好坏，也不提出操作建议。

## 输出形态约定
- 未来输出应结构化，能够映射为 JSON / 卡片等形式，便于下游渲染与追踪。
- 输出内容需可追溯，应指向具体指标、时间段或年份来源，保证引用路径明确。
- 输出需可复制，为研究报告撰写提供可引用的参考语句或要点模板。
- 输出不包含预测、不包含买卖建议，保持描述性与可验证性。

## 明确禁止事项
- 不得给出“好 / 坏 / 买 / 卖”等结论或投资建议。
- 不得预测业绩、股价或其他未来表现。
- 不得引入行业对比、横向排名或同业优劣判断。
- 不得使用情绪化、定性化的判断语言；保持中立、可追溯的事实描述。
